# Stack

Dielabs proprietary frameworks for reasoning about LLM inference systems.

---

## Frameworks

### [Inference Stack Model (ISM)](inference-stack-model.md)
A 7-layer architectural model (L0–L6) that maps every component involved in serving an LLM — from physical hardware to the client application. Used to understand system boundaries, ownership and data flow.

### [Inference Mastery Model (IMM)](inference-mastery-model.md)
A competency framework (Layers A–G) that maps what an inference engineer needs to know and operate. Unlike the ISM, it maps *skills* rather than components — and deliberately cuts across multiple ISM layers.

### [Inference Diagnostic Framework](inference-diagnostic-framework.md)
A structured root-cause analysis methodology built on top of the ISM. Maps observable symptoms (high TTFT, OOM, GPU underutilization) to the layer where they originate.
